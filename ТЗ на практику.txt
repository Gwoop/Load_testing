Необходимо написать программу, которая будет имитировать работу пользователей в БД

1) Параметры связи должны указываться в файле конфигураций. В случае отсутствия данного файла пользователю выдать сообщение с информацией о том что "Необходимо создать файл tnsnames.ora по <путь до проги>"

2) Перед написанием программы проанализировать структуру запросов в базу в программе Search через SQL Monitor. В программе выполнять упрощенный вариант этих запросов. Осуществлять поиск по ID документа.

3) При запуске программы брать содержимое файла \\search\IM\АРМ ТЕХНОЛОГА\ARM\ARMresourse.psw, расшифровывать его значение с помощью CRYPT.dll и подставлять полученные значения пароля в параметры подключения к БД.

4) При запуске программы проверять на наличие в базе искомых таблиц. В случае их отсутствия выдавать сообщение об ошибке "Неверная структура БД. Программа ожидает в файле tnsnames.ora указание на базу со структурой базы Search"

5) Запросы выполнять в несколько потоков. Учитывать загруженность системы и занимать свободных ресурсов не более чем указано в ini-файле. (Например, проверять свободные ресурсы, складывать с занимаемыми программой ресурсами, высчитывать занимаемый %. Если больше указанного числа, гасим несколько потоков. Если меньше, запускаем еще. Чтобы постоянно не гасить/запускать потоки, накидывать к числу +- пару процентов)

6) Программа должна выполняться в фоновом режиме. Для отключения программы должна иметься иконка в трэе.

7) Должна быть возможность настраивать таймеры "сна", на время которых останавливаются все потоки и запросы в базу не посылаются.

8) Если останется время, прикрутить логи. Раз в час указывать количество активных потоков, максимум потоков за прошедший час, выполненных запросов (с момента последнего логирования). Пример строки лога ниже:
26.09.2022 15:41 active_thread: 15, max_thread: 18, sended_request: 150000
26.09.2022 16:41 active_thread: 13, max_thread: 25, sended_request: 180000
